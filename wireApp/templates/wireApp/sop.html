Standard Operating Procedure (SOP): Running the Platform

Objective:

The platform is designed to ensure the robust security of the organisation's network by actively maintaining updated threat intel, leveraging indicators of compromise keyword matching for enhanced threat detection, and employing mechanisms for the identification of malicious communication. Additionally, users can seamlessly download comprehensive reports in the form of facilitating informed decision-making and proactive cybersecurity measures

Preconditions:

  1. Ensure that you have the necessary credentials to access the platform.
  2. The users are generated for System Network Administrators (SANA) of every group.

Procedure:

  1. Login to the Platform:

    • Open a web browser.
    • Navigate to the platform's login page.
    • Enter your credentials (username and password).
    • Click on the "Login" button.
    • Upon successful login user will be redirected to Dashboard

  1. Dashboard:

    • a) My Devices - All computers which are added
      1. Navigate to the "My Devices" section in the dashboard.
      2. View a list of all computers that have been added to your account.
    • b) Add Device - To add computer details
      1. Click on the "Add Device" option in the dashboard.
      2. Follow the steps to provide details and upload necessary files to add a new computer.
      3. Wait for the confirmation message indicating successful addition.
    • c) Reports Section- Download reports for system
    • d) Platform SOP - Instructions to use the platform
    • e) Script SOP- Instructions to run the script
    • f) Download the executable script to run on the system to be inspected

  2. Platform SOP Section:

    • a) Download the audit script
      1. Visit the "Download Audit Script" page in the "Platform SOP" section.
      2. Find the appropriate download link for the audit script.
      3. Click on the link to download the script.
    • b) SOP Document for using the platform
      1. Visit the "SOP Document" page in the "Platform SOP" section.
      2. Access and read the SOP document for guidelines on using the platform.

  3. Additional Tips:

    • Ensure the application has a user-friendly interface for easy navigation.
    • Regularly check for updates in the "Platform SOP" section for any changes or additions to procedures.
    • Contact support if you encounter any issues or need further assistance.

2. Add Computers:

  1. Navigate to the "Add Computers" section in the dashboard.
  2. Click on the "Upload Zip File" button.
  3. Select the zip file containing computer information for upload.
  4. Choose the computer type from the available options (UCS, Internet, Standalone, Server).
  5. Click on the "Submit" or "Upload" button to add the computers.
  6. Wait for the confirmation message indicating successful upload.

3. View Inventory:

  1. Navigate to the "Inventory" section on the dashboard.
  2. A list of all added computers will be displayed, including their types and other relevant details.
  3. Use filters or search options to locate specific computers if necessary.

4. View Computer Details:

  1. From the "Inventory" section, click on the computer's name or ID to view details.
  2. The details page should include information such as computer specifications, status, and any other relevant data.

5. Generate and Download Reports:

  1. Navigate to the "Reports" section on the dashboard.
  2. Select the type of report you want to generate (e.g., Inventory Report, Computer Details Report).
  3. Configure any specific parameters or filters for the report.
  4. Click on the "Generate Report" button.
  5. Once the report is generated, there should be an option to download it as a PDF document.

Troubleshooting:

If you encounter any issues or errors during the process, refer to the platform's help documentation or contact the support team for assistance.

Note:

  • Ensure that you comply with data privacy and security protocols when handling computer details and reports.
  • Regularly update your password and follow security best practices.
